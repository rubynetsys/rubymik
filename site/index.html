RubyMIK is a modern dashboard for MikroTik RouterOS. Health monitoring, live topology, full guarded configuration, config snapshots & restore — and it refuses to let you cut off your own management path. Your router credentials never leave your server.

Everything, in one place

The RouterOS-native depth generic SNMP tools can't cleanly show — for a single box or hundreds across many sites.

▤Fleet monitoring

Status, CPU, memory, uptime, RouterOS version across every site on one screen. Per-interface traffic graphs, DHCP, ARP, routes, wireless, temperature. Auto-refreshing.

◈Live topology

Auto-discovered from MNDP/LLDP/CDP neighbours, drawn with a clean canvas layout that scales to hundreds of devices with level-of-detail.

⚙Full configuration

Firewall, NAT, QoS, DHCP, routes, addresses, bridges/VLANs, PPPoE, VPN, wireless — every write guarded and audited.

◱Snapshots & restore

An automatic config snapshot before and after every change. Diff any change; restore section-by-section — through the same safety guards.

◮Alerting

Device-down, high CPU/memory/temperature, interface-down — debounced, hysteresis, no flapping. Email, webhook, Telegram, WhatsApp.

◆WireGuard remote access

Reach behind-NAT sites over a built-in WireGuard overlay — the router generates its own key; RubyMIK never holds it.

It won't let you lock yourself out

Every configuration write goes through the same pipeline: snapshot → apply → verify the router is still reachable → auto-rollback if not → audit. On top of that, RubyMIK refuses, up front, any change it can prove would sever its own management path.

  • A firewall preset can't drop the management-accept rule — it's always emitted first.
  • A default route or a route over the management subnet is refused before it black-holes you.
  • Management IP / L2 changes use add-before-remove: build the new path, verify, then remove the old.
  • A dead-man watches reachability after every change and reverts within seconds if the path is lost.

Why I built this

I touch MikroTik routers every single day. I love these boxes — but managing a fleet of them means Winbox windows everywhere, and one wrong bridge or VLAN change on a remote site means a drive or a very awkward phone call.

There are other tools out there, and some of them are good. But I never found one that did what I actually needed end to end — full config (firewall, NAT, VPN, queues, PPPoE, bridges/VLANs, DHCP, wireless), automatic config snapshots before and after every change, diffs, restore, updates — and, most importantly, one that refuses to let you cut off your own management path. Every write in RubyMIK goes through guards and a dead-man check, because I've locked myself out of enough routers for one lifetime.

So this started as a fun side project to manage my own routers, and it grew into something I now rely on daily. I'm sharing it because it might help you too.

Honest disclaimer: this is not a perfect app. It's built for my needs, by one person, and there will be rough edges. But it's free, it's self-hosted (your router credentials stay on your server), and it does what it says.
